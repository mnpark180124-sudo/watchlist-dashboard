# 관심종목 모니터 대시보드

23개 관심종목의 실시간 등락률을 보여주는 대시보드.
GitHub Actions가 15분마다 네이버 금융에서 시세를 긁어와 `data/stocks.json`에 저장하고,
GitHub Pages가 그 파일을 읽어서 화면에 그립니다. (서버 없이 정적 페이지 + 자동 커밋 방식)

## 설치 순서 (iPad Codespaces)

1. GitHub에 새 저장소(repo)를 만듭니다. 예: `watchlist-dashboard`
2. 이 폴더의 파일 전체를 그 저장소에 업로드 (또는 Codespaces에서 그대로 붙여넣기)
3. 저장소 **Settings → Actions → General → Workflow permissions** 에서
   **"Read and write permissions"** 로 바꿔줍니다. (Actions가 자동으로 커밋하려면 필요)
4. 저장소 **Settings → Pages** 에서 Branch를 `main`, 폴더는 `/ (root)` 로 설정
5. **Actions** 탭 → `Update Stock Watchlist` 워크플로우 → **Run workflow** 로 한 번 수동 실행해서
   `data/stocks.json`이 실제 시세로 채워지는지 확인
6. 몇 분 뒤 `https://<아이디>.github.io/watchlist-dashboard/` 접속

## 종목 추가/수정

`scraper.py` 맨 위 `WATCHLIST` 리스트에 종목명만 추가하면 됩니다.
자동 검색이 엉뚱한 종목(동명이인 코드 등)을 찾아오면 `CODE_OVERRIDES`에
`"종목명": "종목코드"` 형태로 직접 코드를 적어주세요.
종목코드는 네이버 금융에서 그 종목 페이지 열었을 때 주소창의 6자리 숫자입니다.

## 거래량 급증 · 외국인/기관 수급

`scraper.py`가 각 종목의 오늘 거래량을 최근 20일 평균과 비교해 배율(`volumeRatio`)로 계산하고,
외국인/기관 순매매 수량(`foreignNet`/`instNet`, 최근 거래일 기준)도 같이 저장합니다.
대시보드 타일에는 "거래량 2.4배 · 외국인 +12.0만주 · 기관 -3.2만주" 식으로 표시돼요.
거래량이 평소 2배 이상이면 빨간 글씨로 강조됩니다.

이 값들은 네이버의 일별시세/외국인·기관 매매동향 표를 컬럼명 기준으로 찾아서 읽어옵니다.
페이지 구조가 바뀌면 `fetch_volume_surge`, `fetch_foreign_institution` 함수를 확인해보세요.

## 매크로 지표 (코스피/코스닥/VKOSPI/환율/나스닥/SOX)

대시보드 상단에 코스피, 코스닥, VKOSPI(변동성지수), 원/달러 환율, 나스닥, 필라델피아반도체지수(SOX)를
한 줄로 보여줍니다. 국내 지수(코스피/코스닥/VKOSPI)는 네이버 지수 페이지에서, 해외 지표(환율/나스닥/SOX)는
`yfinance`로 가져옵니다.

**VKOSPI 관련 주의**: 네이버의 VKOSPI 지수 코드가 바뀌었을 수 있어서, 값이 계속 `-`로 나오면
`scraper.py`의 `fetch_naver_index("VKOSPI", ...)` 호출 부분에서 코드값을 확인해야 할 수 있어요.
네이버 금융에서 "VKOSPI" 검색 후 나오는 페이지 URL의 `code=` 파라미터를 그대로 넣어주면 됩니다.

## 평가점수 · 52주 최고/최저 · 증권사 투자의견

`scraper.py`가 시세와 함께 네이버 금융 페이지에서 52주 최고/최저가, 목표주가 컨센서스,
투자의견(매수/중립/매도)도 같이 긁어와서 `data/stocks.json`에 저장합니다.
대시보드는 이 값들과 뉴스 영향도(`data/news.json`)를 조합해서 종목마다 "평가점수"를
브라우저에서 즉석으로 계산해 보여줍니다. 가중치는 `index.html`의 `SCORE_WEIGHTS`에서
바로 조절할 수 있어요 (기본: 목표주가 상승여력 35%, 투자의견 25%, 52주 위치 25%, 뉴스 15%).

**중요**: 이 점수는 매수/매도 추천이 아니라 여러 지표를 단순 가중합한 참고용 수치예요.
52주 저점에 가까울수록 점수가 높게 나오는 건 "저평가일 수 있다"는 관점 하나를 반영한 것뿐이고,
반대로 고점 돌파 후 상승 추세를 좋게 보는 관점도 있을 수 있으니 절대적인 기준으로 쓰지 마세요.

네이버 페이지에서 라벨 텍스트("52주최고", "목표주가", "투자의견")를 기준으로 값을 찾기 때문에,
페이지 구조가 크게 바뀌면 해당 필드만 `null`로 채워지고(에러는 안 남) 점수 계산에서 자동으로
빠집니다. 값이 계속 비어있으면 `scraper.py`의 `fetch_extra` 함수를 확인해보세요.

## 뉴스 영향도 분석 (선택 기능)

`news_analyzer.py`는 Gemini API(Google 검색 그라운딩 사용)로 종목별 전날 국내/해외 뉴스를 검색해서
중요 뉴스가 있으면 요약과 "영향도 추정치(%)"를 만들어 `data/news.json`에 저장합니다.
대시보드는 이 파일을 같이 읽어서 각 종목 타일 아래에 항상 뉴스 요약(또는 "특이 뉴스 없음")을 보여줘요.

**사용하려면:**
1. [aistudio.google.com](https://aistudio.google.com)에서 구글 계정으로 로그인 → **Get API key** → **Create API key** (신용카드 불필요)
2. 저장소 **Settings → Secrets and variables → Actions**에 `GEMINI_API_KEY`로 등록
3. `.github/workflows/news.yml`이 하루 2번(장전 08:30, 장후 16:00 KST) 자동 실행

**비용**: `gemini-2.5-flash-lite` 모델은 무료 티어에서 하루 500회 요청 + 검색 그라운딩을 지원해서,
23종목 × 하루 2번(46회) 정도는 무료 범위 안에서 돌아가요. 다만 구글 무료 티어 한도는
정책에 따라 바뀔 수 있으니, Actions 탭에서 실행 로그를 가끔 확인해보시는 게 좋아요.
한도를 초과하면 그 회차는 실패하고 다음 스케줄에 다시 시도됩니다.

**꼭 알아두실 점**: 영향도(%)는 실제 주가 예측치가 아니라 Gemini가 뉴스를 읽고 판단한
주관적 추정치예요. 참고용으로만 쓰시고, 이 수치만 보고 매매 결정을 내리지 않는 걸 권장해요.

## 주의할 점

- 네이버 금융 API는 비공식 API라 언제든 응답 형식이 바뀔 수 있어요.
  스크래퍼가 갑자기 안 돌아가면 `fetch_price` 함수의 파싱 부분을 먼저 의심해보세요.
- 장 마감 후·주말에는 크론이 안 돌게 해뒀습니다(평일 09:00~15:30 KST, 15분 간격).
  더 자주/늦게까지 갱신하고 싶으면 `.github/workflows/update.yml`의 cron 값을 조정하세요.
